## コンセプト
自分の体験したトキメキ（自慢したいこと/うれしかったこと）を投稿できるサイト

## 対象ユーザ
- 日常にキラキラを足したい人
- 自慢話をほかの人に言いにくい人

## 利用の流れ
メールアドレス+パスワードで新規登録/ログイン

タイムラインでほかの人の投稿を見る
- 良いと思ったもの/好きな投稿を保存する
- 保存した投稿のみを表示する

新規作成から575(57577)の投稿をする

## 推しポイント
575もしくは57577の文字数制限

いいね機能なし

他の人のキラキラ話を見てうれしい気持ちに！

## 開発体制

### 役割分担
伊藤：リーダー/バックエンド/フロントエンド

川田：UIデザイン/発表資料作成

布目：フレームワークデザイン/プロダクトロゴ作成/動画編集

澤田：フロントエンド/デザイン補助

早川：フロントエンド/UIデザイン

### 開発における工夫した点
チーム開発
- できるだけ対面で、その時集まれる全員で作業を進めた
- 意見をすぐ共有する/文字や記録として残す
- たくさんご飯に行って仲良くなる！
- なんでも褒めあう

技術
- バックエンドの経験者がいないため、サーバー管理が不要なfirebaseを用いた
  - フロントエンドの実装に集中できた
  - メールアドレスでの簡易登録やログイン状態の保持に＋αで知識を付ける必要がないため、ログイン機能を実装できた
- フロントエンドにVueを用いた
  - 1,2年生の授業で扱うHTML、CSSを応用して書くことができた
  - そのうえで、NUXT.jsへのチャレンジができた

スケジュール/タスク管理
- Discord上のリアクションで各自空いている時間を共有
- NotionのTodoリストでタスク/担当管理→Discordにもスクリーンショットを共有
- 徹夜なし！
- GitHubのIssueですべきことの順序立てを効率的に行えた
  - リモートでの作業もスマート
